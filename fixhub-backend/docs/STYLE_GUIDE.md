# 文档与注释规范（简体中文）

为统一项目风格、便于团队协作与维护，本仓库采用以下文档和注释规范：

- 所有对外或开发者可见的文档（README、ARCHITECTURE、设计文档、API 文档）使用简体中文编写。
- Java 源代码文件中的注释（类注释、方法注释、复杂逻辑注释）一律使用简体中文。
- 当需加入示例代码或配置片段（JSON、YAML、SQL、Shell）时，代码本身保持原语法，但上下文描述与说明使用中文。
- 提交 PR 时，如果修改或新增文档，务必在 PR 描述中使用中文说明变更要点。

建议：使用简洁、规范的语言，尽量使用完整句子，避免遗漏必要细节。如需引用英文原文或外部链接，可在括号中附上英文或链接供参考。
