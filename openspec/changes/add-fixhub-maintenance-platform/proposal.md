# Change: 添加设备报修与维护平台核心能力

## Why
课程要求建设一个面向学校/部门的在线报修平台，覆盖角色管理、报修流程、状态跟踪、设备管理与数据看板，并提供清晰可读的接口文档。

## What Changes
- 新增角色与权限能力（报修用户、维修工、管理员），并限定关键操作权限。
- 新增报修流程能力：提交、派单、接单/处理、评价与关闭。
- 新增工单状态跟踪与查询能力（待受理、处理中、待评价、已完成）。
- 新增设备管理能力（管理员维护公共设备信息）。
- 新增数据看板统计能力（维修量、响应时长、满意度）。
- 完善接口文档与中文提示，确保对外文档可读且一致。

## Impact
- Affected specs: roles, repair-orders, device-management, dashboard-analytics, api-docs
- Affected code: 后端控制器/服务/数据模型/数据库脚本；前端页面/接口调用；文档与 OpenAPI 描述
